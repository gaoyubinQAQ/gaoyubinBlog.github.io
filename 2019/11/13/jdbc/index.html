1.什么是JDBC？

JDBC（Java DataBase Connectivity）就是Java数据库连接，说白了就是用Java语言来操作数据库。

​ 原来我们操作数据库是在控制台使用SQL语句来操作数据库，JDBC是用Java语言向数据库发送SQL语句。

2.驱动！

​ JDBC是接口，而JDBC驱动才是接口的实现，没有驱动无法完成数据库连接！每个数据库厂商都有自己的驱动，用来连接自己公司的数据库。

​ 我们要链接哪个数据库 就需要导入相关数据库的jar包！

3.JDBC核心类的介绍

​ JDBC中的核心类有：DriverManager、Connection、Statement，和ResultSet！

​ DriverManager——-驱动类 加载驱动 此类中有个方法可以提供连接数据库的通道！如过能通过此类得到通道，那么证明数据库连接 成功！

​ 静态方法 DriverManager.getConnection(“链接协议（路径）”,”用户名”,”密码”);可以返回链接数据库的Connection对象

​

​ Connection——–通道类 从DriverManager中获取的就是此类的对象！这是一个连接数据库的管道，让我们可以通过它去连接数据库，然后操作数据库！

​

​

​ Statement——–操作类 操作数据库的对象，此类的对象可以通过通道对数据库的数据进行操作（添删改查）！依赖于通道类！（通过通道获取此对象，也可以理解为通道提供的一个连接方式）

​

​

​ ResultSet——-结果集对象 操作了数据库，取出来的临时数据类，此时数据库里的数据会被存在此类里面，可以通过此类的get….方法去获取数据（只有在执行查询的时候才有这个对象）

​

二、JDBC实际操作

​ 1.第一步，建立工程

​ 新建一个工程（这里只测试JDBC，所以只需要建一个JAVA工程就可以，之后要跟前台交互，再建WEB工程）

​

​ 导包：把MySql的包引入！

​ 2.第二步，加载驱动

​ 只有一句话，Class.forName(驱动类);，这里我们用的是MySql的驱动：”com.mysql.jdbc.Driver”——-com.mysql.jdbc下的Driver类

​ 3.第三步，获取通道Connection

​ 上面说到 驱动加载完成之后可以通过DriverManager类来获取连接数据库的通道 具体代码：Connection con = DriverManager.getConnection (“jdbc:mysql://localhost:3306/mydb1”,”root”,”123”);

​ 4.第四步，获取操作类对象Statement

​ 通过通道类对象获取创建，上面说了，不做多余表述。具体代码：Statement sta = con.createStatement();

​ 5.第五步，执行添删改操作

​ 通过操作对象可以对数据库数据进行添加更改删除操作，由于不获取数据，所以用不到ResultSet结果集对象！具体代码：sta.execute(SQL语句);注意 执行修改操作时方法为：sta.executeUpdate(修改的SQL语句);

​

​ 6.第五步，执行查询操作

​ 执行查询操作的时候需要获取数据，这时候就要用到 ResultSet——-结果集对象了

​ 代码：ResultSet res= sta.executeQuery（SQL语句）;

​ ResultSet 对象有个方法叫res.next();返回类型是布尔类型，表示此结果集有没有下一个，可以用来做循环，吧所有数据都取出来！代码如下：

​ while(res.next()){

​

​ String sname=res.getString(“Sname”);//此方法可以get各种数据类型

​ }

​

三、预处理

什么是预处理

​ 也叫作预编译

​ 一、数据库在执行一条语句的时候的过程：

​ 1.先判断sql语句的语法是否正确

​ 2.判断所传值的类型是否正确

​ 3.执行 返回结果

​ 二、而我们在需要经常性多次的操作同一条sql语句的时候，以上三种操作都会重新执行一遍，

​ 其中第一条判断语法是否正确的操作一次次执行减缓效率（因为是同一条语句）

​ 三、预处理就是先把准备好的sql语句提前编译，也就是执行了第一条，确定了语法没有问题之后，再赋值给我们的参数，在下一次执行的时候，

​ 只执行2.3条就可以，节省资源，加快效率

​

预处理的好处

​

​ 一、效率高！

​ 二、安全性好（防止sql注入）

​ sql注入是指sql语句拼接的时候的不安全性

​ 例如：select * from users where username=参数1 and password=参数2 而此时我若是把参数1传一个 ‘a’ or 1=1 参数2为 ‘b’ or 1=1

​ 拼接结果 SELECT * FROM users WHERE username=’a’ OR 1=1 AND PASSWORD=’b’ OR 1=1

​

预处理的应用

​ 一、sql语句可以把需要拼接的参数用？代替

​ 例如：update commodity set COMMODITY_NAME=?, COMMODITY_PRICE=? WHERE ID=?

​ 二、我们在获取操作对象的时候 不再使用Statement 而是使用PreparedStatement

​ PrepareStatement就是预处理对象，他继承 Statement

​ 代码：

​ PrepareStatement ps = con.prepareStatement(sql);//ps为预处理对象 con.prepareStatement(sql)为链接对象编译sql语句 返回预处理对象

​ 三、给参数赋值

​ ps.setXXX(int index,Object value);方法需要两个参数 (与set类型匹配)

​ 1.第一个参数int类型的index,代表要为为sql语句中需要赋值的第几个参数（即第几个？符号）

​ 2.第二个参数value 代表要给当前参数赋的值 注意：一定要保证顺序和值的正确，否则有可能造成数据错乱或者出错

​ 示例：

​ ps.setString(1,”COMMODITY_NAME的值”);

​ ps.setDouble(2,”COMMODITY_PRICE的值”);

​

四、事务

什么是数据库中的事务

​ 在我们操作数据时，有时候是需要把两件事情绑定到一起去做的，要么全部成功，要么全部失败，不允许发生一件事情成功而另一件事情失败的现象

​ 例如：银行转账！张三转10000块到李四的账户，这其实需要两条SQL语句：

​  给张三的账户减去10000元；

​  给李四的账户加上10000元。

​ 如果在第一条SQL语句执行成功后，在第二条SQL语句执行完成之前，程序被中断了（可能是抛出了某个异常，也可能是其他什么原因），

​ 那么李四的账户没有加上10000元，而张三却减去了10000元。这肯定是不行的！

​ 为了保证数据的准确，完整性，发明了事务

事务的四大特性

​ 1.原子性：事务中所有操作是不可再分割的原子单位。事务中所有操作要么全部执行成功，要么全部执行失败。

​ 2.一致性：事务执行后，数据库状态与其它业务规则保持一致。如转账业务，无论事务执行成功与否，参与转账的两个账号余额之和应该是不变的。

​ 3.隔离性：隔离性是指在并发操作中，不同事务之间应该隔离开来，使每个并发中的事务不会相互干扰。

​ 4.持久性：一旦事务提交成功，事务中所有的数据操作都必须被持久化到数据库中，即使提交事务后，数据库马上崩溃，在数据库重启时，也必须能保证通过某种机制恢复数据。

事务在数据库中的操作

​ 以mysql为例：开启事务的sql语句：start transaction;这句话表示开启了事务，在事务关闭之前的所有操作都将被视为同一事务。

​ 关闭事务的sql语句：commit或rollback commit代表提交事务，保存 rollback表示所有操作回滚到开启事务之前的状态

​ 示例： start transaction;//开启事务

​ update account set money=money+1000 where name=’张三’;//执行事务中第一条语句

​ update account set money=money-1000 where name=’李四’;//执行第二条语句

​ commit;//提交事务 事务结束 或者rollback;//回滚事务 恢复数据

事务在JDBC中的操作

​ 事务在JDBC中都是用Connection对象来完成的 Connection对象的三个方法与事务相关：

​ 1.setAutoCommit(boolean);设置是否为自动提交事务，默认为true表示自动提交，也就是每条执行的SQL语句都是一个单独的事务，如果设置false，那么就相当于开启了事务了!

​ \2. commit();提交结束事务；

​ \3. rollback();回滚结束事务。

​ jdbc处理事务的代码格式：

​ try {

​ con.setAutoCommit(false);//开启事务…

​ ….给张三+100

//抛出异常

​ …给李四-100

​ con.commit();//try的最后提交事务

​ } catch() {

​ con.rollback();//回滚事务

​ }

​ JDBC事务的补充：同一事务必须用的是同一个Connection对象！
